# Submission Checklist

> Use this as a pre-submission verification list before March 18, 2026 5:00pm EDT

## Required Submissions

### 1. Code Repository
- [ ] Public GitHub/GitLab repository
- [ ] Open source license file included (MIT, Apache 2.0, etc.)
- [ ] License visible in repo's "About" section
- [ ] README with clear setup/run instructions
- [ ] All source code, assets, instructions included
- [ ] No secrets/API keys committed
- [ ] Code remains public post-event

### 2. Text Description
- [ ] Clear explanation of features and functionality
- [ ] **Detailed description of DigitalOcean Gradient AI usage**
- [ ] Target audience/use case described
- [ ] Tech stack listed
- [ ] Written in English

### 3. Demo Video
- [ ] Under 3 minutes
- [ ] Shows project actually running (not slides/mockups)
- [ ] Uploaded to YouTube/Vimeo/Facebook Video (public)
- [ ] No copyrighted music or unauthorized trademarks
- [ ] Clearly demonstrates Gradient AI integration
- [ ] Shows full user flow from start to finish
- [ ] English or with English subtitles

### 4. Gradient AI Usage Description
- [ ] Which Gradient AI features were used
- [ ] How they integrate into the application
- [ ] Why Gradient AI was the right choice
- [ ] Architecture diagram showing DO services used

## Optional but Recommended

### 5. Live Demo URL
- [ ] Functional application accessible to judges
- [ ] Login credentials included if needed (in submission form)
- [ ] Stable and responsive during judging period (Apr 7-14)

### 6. Architecture Documentation
- [ ] System architecture diagram
- [ ] Data flow diagram
- [ ] DigitalOcean services integration map

## Pre-Submission Verification

### Technical
- [ ] Application installs and runs successfully
- [ ] All features demonstrated in video actually work
- [ ] No hardcoded credentials or environment-specific paths
- [ ] Application works on intended platform

### Content
- [ ] All text in English
- [ ] Video link is public and accessible
- [ ] Repository URL is correct and accessible
- [ ] Devpost submission form completely filled out

### Legal
- [ ] Project is original work created during submission period
- [ ] No re-used code from before hackathon
- [ ] Third-party integrations properly licensed
- [ ] OSI-approved license included
