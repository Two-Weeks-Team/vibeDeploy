## Stage 1: Pass/Fail Gate

Before scoring, judges check:
- Does it fit the theme? (AI application)
- Does it use Gradient AI? (Required APIs/SDKs)

**If you don't clearly show Gradient AI usage, you're eliminated before scoring even begins.**

---

## Maximizing Each Criterion

### 1. Technological Implementation (25%)

**Goal:** Show that Gradient AI is the CORE of your app, not an afterthought.

**Checklist for maximum points:**
- [ ] Use **Serverless Inference API** (chat completions, image generation)
- [ ] Implement **Knowledge Bases** (RAG with auto-indexed documents)
- [ ] Build with **Agent Development Kit (ADK)**
- [ ] Add **Guardrails** (content moderation, PII detection)
- [ ] Show **Agent Evaluations** (quality metrics)
- [ ] Display **Tracing** in the architecture
- [ ] Use **multiple DO services** (App Platform + DB + Spaces + Gradient)
- [ ] Clean code architecture (well-organized, documented)
- [ ] Error handling, logging, security best practices
- [ ] Open-source with proper OSI license

### 2. Design (25%)

**Goal:** Professional-quality UI that any user can navigate intuitively.

**Checklist:**
- [ ] Modern UI framework (Next.js, React, etc.)
- [ ] Consistent design system (colors, typography, spacing)
- [ ] Mobile-responsive
- [ ] Loading states, error states, empty states
- [ ] Smooth animations and transitions
- [ ] Clear user flow (onboarding → core feature → result)
- [ ] Accessibility basics (contrast, alt text, keyboard nav)
- [ ] Dark mode (bonus polish)

### 3. Potential Impact (25%)

**Goal:** Solve a real problem that affects a large or underserved community.

**Strong impact narratives:**
- Developer productivity (tooling)
- Education accessibility
- Open-source community benefit
- Healthcare/wellness
- Environmental/sustainability
- Small business/startup enablement

**Checklist:**
- [ ] Clearly defined target audience
- [ ] Quantifiable problem statement
- [ ] AI meaningfully improves the solution (not just slapped on)
- [ ] Scalability story (how it grows)
- [ ] Potential user numbers or market size

### 4. Quality of the Idea (25%)

**Goal:** Stand out from 2,000+ submissions with originality.

**Avoid overdone ideas:**
- Generic chatbot
- Simple summarizer
- Basic image generator frontend
- Another "chat with your docs"

**Stand out with:**
- AI applied to an unexpected domain
- Novel agent architecture (multi-agent collaboration)
- Creative use of Gradient AI's unique features
- Solving a problem most people haven't thought about
- Technical innovation in how AI is applied

## The Video (MOST IMPORTANT DELIVERABLE)

> Judges may judge **solely** on text description, images, and video.
> They are NOT required to test the project.

**This means the video IS your product.**

### vibeDeploy Video Formula (under 3 minutes):

```
0:00 - 0:10  │  Hook: "What if building an app was as easy as one sentence?"
0:10 - 0:25  │  User pastes YouTube URL or types idea
0:25 - 1:00  │  LIVE: Vibe Council Meeting — 6 agents debate in real-time (visual)
1:00 - 1:15  │  Vibe Score reveal: "82/100 — GO!" (gauge animation)
1:15 - 1:35  │  AI generates code (split: frontend + backend + logo generation)
1:35 - 2:00  │  Deploy to DO → Live URL → click → actual app works
2:00 - 2:20  │  Architecture diagram: 8 Gradient AI features shown
2:20 - 2:40  │  "Open source, for everyone" + GitHub link
2:40 - 3:00  │  "From idea to live app. Powered by DigitalOcean Gradient AI."
```

### 킬러 와우 모먼트 (3개)
1. **Vibe Council 라이브 토론** — 6개 AI가 서로 도전하고 반박하며 토론하는 모습 (유일무이)
2. **Vibe Score 게이지 애니메이션** — 5축 점수가 실시간으로 차오르는 연출
3. **라이브 URL 클릭** — 생성된 앱이 실제로 동작 (가장 강력한 증거)

### Video Tips:
- **Show, don't tell** — always show the app running
- **Highlight Gradient AI usage explicitly** — call it out by name
- **Professional audio** — clear narration, no background noise
- **Screen recording quality** — HD, smooth, no lag
- **Architecture diagram** — visual showing DO services integration

## Submission Description (Draft)

```markdown
## vibeDeploy — From Idea to Live App in One Conversation

### Problem
84% of developers use AI coding tools, but getting from "vibe-coded" app to production
remains a painful, manual process. The gap between "AI-generated code" and a deployed,
working application requires DevOps expertise most people don't have.

### Solution
vibeDeploy takes a single sentence or YouTube URL and convenes The Vibe Council — a live
AI meeting with 6 specialized agents (Architect, Scout, Guardian, Catalyst, Advocate, and
Strategist). They debate the idea through cross-examination, score it on 5 dimensions
(Vibe Score), and if feasible, automatically generate a PRD, codebase, AI-designed logo,
and deploy a live MVP to DigitalOcean — complete with a working URL.

### DigitalOcean Gradient AI Usage (8 features)
- **Agent Development Kit (ADK)**: Entire multi-agent system built and deployed via ADK
- **Serverless Inference (Text)**: GPT-5.3-Codex (orchestration), Claude Sonnet 4.6 (code gen),
  GPT-5 mini (Vibe Council agents), GPT-5 (cross-examination)
- **Serverless Inference (Image)**: GPT-image-1 (app logos), Fast SDXL (placeholder images),
  Flux Schnell (OG social cards)
- **Knowledge Bases (RAG)**: DO documentation + framework best practices indexed for
  accurate deployment config generation
- **Multi-Agent Routing**: LangGraph StateGraph orchestrating 6 Vibe Council agents with parallel Send API
- **Agent Evaluations**: Correctness, Instruction Following, Context Adherence metrics
- **Agent Tracing**: Full pipeline visibility — every agent step traced automatically
- **Function Calling**: DO API (deployment), GitHub API (repo creation), YouTube API (transcript)

### Additional DigitalOcean Services
- **App Platform**: vibeDeploy hosting + user app deployment target
- **Managed PostgreSQL**: Session storage + user app databases
- **Spaces Object Storage**: Generated code, images, documents

### Architecture
User Input → Input Processor → Vibe Council Meeting (6 agents debate) → Vibe Score (5 axes) →
Decision Gate → Doc Generator → Code Generator → Deployer → Live URL

### Tech Stack
- Frontend: Next.js 15 + shadcn/ui + Tailwind CSS
- Agent Backend: Python 3.12 + Gradient ADK + LangGraph
- AI: DigitalOcean Gradient AI (text + image inference)
- Database: DigitalOcean Managed PostgreSQL 16
- Storage: DigitalOcean Spaces
- Deployment: DigitalOcean App Platform

### Impact
- **Target**: Non-technical founders, solo developers, hackathon participants
- **Problem Scale**: 84% of developers use AI tools, deployment remains the bottleneck
- **Vision**: Democratize software creation — anyone with an idea can ship a live app
- **Open Source**: MIT licensed, community-driven

### Future Plans
- Support more frameworks (Vue, Svelte, Django, Rails)
- Team collaboration features
- Custom domain setup automation
- Generated app monitoring dashboard
- Mobile app generation (React Native)
```
